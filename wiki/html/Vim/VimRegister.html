寄存器(Registers) :reg : 列出寄存器, 显示所有当前的register

  1. 附加到一个register (就是用大写的register名字啦！) VimRegister

"a5yy #复制5行到a中 10j #下移10行 "A5yy #再添加5行到a中

yy@"

"" - The unnamed register. "d", "c", "s", "x" 和 "y" 等涉及删除或拷贝的命令都会将内容存放于此(有例外)。 @" - run the content in register " as a command yy@" - We can use yy@" to run the current line's content as a command "ayy@a - store the current line in register a as a command and execute it.

# 寄存器的主要有以下几种：

"" - The unnamed register. "d", "c", "s", "x" 和 "y" 等涉及删除或拷贝的命令都会将内容存放于此(有例外)。

"0 to "9 - 10 numbered registers. 0 为最近拷贝的内容，1-9 为最近删除的内容。指定 named register 时不会存放于此。 "- - The small delete register 删除不多于一行的内容会存在于此。 "a to "z "A to "Z - 26 named registers 小写与大写的区别是，小写是覆盖方式，大写是添加方式。 e.g 附加到一个register (就是用大写的register名字啦！)

"a5yy #复制5行到a中 10j #下移10行 "A5yy #再添加5行到a中

":, "., "% and "# - four read-only registers ": 上次执行的命令，冒号就是命令行提示符嘛。这个有些什么用呢？在测试配置文件时十分有用，先在命令行输入命令，调试好后，在 insert mode 里按Ctrl-R : 就可将该命令插到文件中。 ". 存放上次插入的文本，例如，你输入了一段文本，在另一个地方也要有相同的内容，你输完后，一般要选择并复制一下吧？而用这个，直接移到需要插入的地方，i<ctrl-r>. 即可。 "% 是当前编辑文件全名，vim 中，% 就是代表当前编辑文件名的变量，好记吧？不过，一般直接用 % 就行，例如":!cat %"，vim 会自动扩展。 "# 是alternate file 名称 (这个我不知道)

"= - the expression register 呵，表达式寄存器，可以输入一个表达式，并插入结果。

"*, "+ and "~ - The selection and drop registers 跟 gui 即 X/Windows 剪贴板有关的变量。在 X 中，鼠标中键与系统剪贴板不一样，至于区别，大家自己试验一下，这里给个提示，在命令行下输入 reg 可以列出所有寄存器的内容。 "* 关联系统选择缓冲区 "+ 关联系统剪切板

"_ - The black hole register 黑洞

"/ - Last search pattern register 上次搜索的内容。例如，我要搜索一个单词 linuxforum，但我比较懒，先输入 /linux，发现出现很多个 linux，多次按 n 无果，难道要重新输入 /linuxforum ? 不用，只需要 /<ctrl-r>/forum 即可。呵，聪明的你一定想到了，用命令行历史也可以调来，按 ctrl-p 或 ctrl-n 即可上下翻阅。这样也可以嘛，也比较快捷，但要查找 forumlinux 呢？用 ctrl-r 的话只需 /forum<ctrl-r>/ 。

:reg : 列出寄存器, 显示所有当前的register "1p : paste from register 1 "表示引用register，1表示一个名字叫做1的register

  1. 译释："也用来定义register
  2. 先输入 ",表示定义register
  3. 然后输入名字，如0~9,a~z
  4. 然后执行删除或复制命令，如dd或y，
  5. 或者是visual模式下的d(删除选中的部分)或y(复制选中的部分)
  6. 则被删除或复制的部分就被存入了这个命名的register

#

  1. 观察：一个特殊的register, "" ，里面存储了一个匿名的删除/复制
  2. 在你执行dd或y的时候，被作用的部分被存到了""中
